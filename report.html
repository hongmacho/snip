Architecture

Repository Pattern

The project implements the Repository pattern to encapsulate data access operations:

  • SnippetRepository: Manages all database operations for snippets with methods: create, get_all, get_by_id, search, delete, update, get_stats
  • Snippet Model: Represents a single code snippet with attributes: id, title, language, tags, description, code, created_at, updated_at
  • SQLite Database: Local persistent storage with FTS5 full-text search capability

Database Schema

CREATE TABLE snippets ( id INTEGER PRIMARY KEY AUTOINCREMENT, title TEXT NOT NULL, language TEXT NOT NULL DEFAULT 'text', tags TEXT DEFAULT '', description TEXT DEFAULT '', code TEXT NOT NULL, created_at TEXT DEFAULT CURRENT_TIMESTAMP, updated_at TEXT DEFAULT CURRENT_TIMESTAMP ); CREATE VIRTUAL TABLE snippets_fts USING fts5( title, description, code, content=snippets, content_rowid=id );

Usage Examples

Installation

$ git clone https://github.com/hongmacho/snip.git $ cd snip $ pip install -e ".[dev]"

Add a Snippet

$ snip add --title "Python Loop" --language python \ --description "Example for loop" --tags "python,loops" Enter code (press Ctrl+D when done): paste your code ✓ Snippet added with ID 1

List Snippets

$ snip list $ snip list --language python $ snip list --tag loops

Search Snippets

$ snip search "def function"

Copy to Clipboard

$ snip copy 1 ✓ Snippet #1 copied to clipboard

View Statistics

$ snip stats

Export and Import

$ snip export --output backup.json $ snip import --file backup.json
