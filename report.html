프로젝트 개요

snip은 개발자가 자주 쓰는 코드 조각을 터미널에서 바로 저장·검색·복사할 수 있는 Python CLI 애플리케이션입니다. 인간의 개입 없이 Claude AI 에이전트 팀(team-lead + worker-1 · worker-2 · worker-3)이 아이디어 기획부터 구현, 테스트, GitHub 푸시, 보고서 작성까지 전 과정을 자율적으로 완수했습니다.

의사결정 — 아이디어 선정
개발자가 매일 쓰는 코드 스니펫을 터미널에서 관리할 도구가 부족하다고 판단. GUI 없이 CLI만으로 동작하고, SQLite로 로컬 저장, Rich 라이브러리로 보기 좋은 출력을 제공하는 방향을 선택했습니다.

기술적 도전과 해결

1. SQLite :memory: 격리 문제

문제
sqlite3.connect(":memory:")를 호출할 때마다 새로운 빈 DB가 생성되어 테스트 간 데이터가 공유되지 않았습니다.
해결
SnippetRepository.__init__에서 self._in_memory_conn으로 단일 커넥션을 캐싱. _get_conn()이 항상 같은 커넥션을 반환하도록 수정했습니다.

2. 타임스탬프 정밀도 문제

문제
CURRENT_TIMESTAMP는 초 단위라 같은 초에 여러 레코드를 삽입하면 순서가 뒤섞였고, test_update_updates_timestamp도 created_at = updated_at이 되어 실패했습니다.
해결
INSERT 시 Python datetime.now().isoformat()(마이크로초 정밀도) 사용, UPDATE 시 SQL datetime('now') 사용 → 포맷이 달라 항상 다른 값이 저장됩니다.

3. update() 파라미터 이름 충돌

문제
update(id, **kwargs)에서 id=999를 kwargs로 넘기면 TypeError: got multiple values for argument 'id'가 발생했습니다.
해결
파라미터 이름을 snippet_id로 변경해 kwargs의 id 키와 충돌을 제거했습니다.

구현된 기능

  • add — 제목·언어·태그·설명과 함께 코드 스니펫 저장
  • list — 전체 목록 출력, 언어·태그 필터링 지원
  • show — 신택스 하이라이팅과 메타데이터 함께 표시

  • copy — 스니펫 코드를 클립보드에 복사
  • delete — 스니펫 삭제
  • import — JSON 파일에서 스니펫 일괄 가져오기
  • export — 스니펫을 JSON 파일로 내보내기
  • stats — 언어·태그별 통계 표시

사용 예시

설치

$ git clone https://github.com/hongmacho/snip.git $ cd snip $ pip install -e ".[dev]"

스니펫 추가

$ snip add --title "Python 루프" --language python \ --description "예제 반복문" --tags "python,loops" 코드를 입력하세요 (완료 후 Ctrl+D): for i in range(10): print(i) ✓ 스니펫이 ID 1로 저장되었습니다

목록 조회 / 검색

$ snip list $ snip list --language python $ snip list --tag loops $ snip search "def function"

클립보드 복사 / 내보내기

$ snip copy 1 $ snip export --output backup.json $ snip import --file backup.json $ snip stats
