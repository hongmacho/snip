검증된 기존 스니펫

snip은 터미널에서 코드 스니펫을 저장·검색·복사하는 Python CLI 도구입니다. 이 작업에서는 기존 CLI 코드베이스를 유지한 채 FastAPI 백엔드 + Vanilla JS SPA 웹 애플리케이션으로 확장하였습니다. 핵심 저장소 레이어(snip/db.py, snip/repository.py, snip/models.py)는 수정 없이 그대로 임포트하여 CLI와 웹이 동일한 SQLite 데이터베이스를 공유합니다.

프로젝트 목표

  • CLI 전용 기능을 브라우저에서 완전히 사용 가능하게 구현
  • 빌드 도구 없이 제로 의존성 프론트엔드 제공
  • 기존 SQLite 데이터를 웹과 CLI가 공유
  • 코드 신택스 하이라이팅 및 클립보드 복사
  • 실시간 검색 및 언어·태그 필터

🔌 REST API 엔드포인트

GET / index.html SPA 진입점 (FileResponse)
GET /api/snippets 전체 목록 — 쿼리: language, tag
POST /api/snippets 스니펫 생성 (201) — body: title, language, code, tags, description
GET /api/snippets/search 전문 검색 — 쿼리: q (title·description·code 대상)
GET /api/snippets/{id} 단건 조회 — 없으면 404
PUT /api/snippets/{id} 부분 업데이트 (Pydantic Optional 필드)
DELETE /api/snippets/{id} 삭제 — 없으면 404, 성공 시 {"ok": true}
GET /api/stats 통계 — total, languages{}, tags{}

실제 API 응답 예시

{"total": 6, "languages": {"bash":3, "python":2, "py":1}, "tags": {"docker":1, "git":1, ...}}

// GET /api/snippets — 항목 구조

 "tags":"docker,cleanup", "description":"...", "code":"...",

🖥 프론트엔드 구조

index.html — UI 구성

  • 헤더: 검색창 (실시간) + New Snippet 버튼
  • 사이드바: 언어 필터 / 태그 필터 / 통계
  • 메인 그리드: 스니펫 카드 목록
  • 상세 모달: highlight.js 코드뷰 + 복사/삭제
  • 폼 모달: 새 스니펫 추가 (언어 셀렉트)
  • 토스트: 성공/오류 알림 (3초 자동 소멸)

app.js — 핵심 함수

  • normalizeSnippet(): tags 문자열 → 배열 변환
  • fetchSnippets(lang, tag): 필터 포함 목록 조회
  • searchSnippets(q): 전문 검색 (300ms debounce)
  • renderSnippetList(): 카드 DOM 생성 + 이벤트 바인딩
  • renderFilters(): 현재 스니펫에서 언어/태그 동적 추출
  • showDetail(): 모달 + hljs.highlightElement()
  • escapeHtml(): XSS 방지

태그 정규화 — normalizeSnippet()

// DB에서 "git,revert" 문자열로 오는 tags를 배열로 정규화
function normalizeSnippet(s) {
    return {
        ...s,
        id: String(s.id),
        tags: s.tags ? s.tags.split(',').map(t => t.trim()).filter(Boolean) : []
    };
}

🔧 발견 및 수정된 버그

API 응답 래퍼 불일치 — fetchSnippets / searchSnippets
app.js가 data.data 래퍼를 기대했으나 FastAPI는 배열을 직접 반환.
→ Array.isArray(data) ? data : []로 수정 후 normalizeSnippet 적용.
stats 필드명 불일치 — renderStats()
stats.total_snippets → stats.total,  stats.by_language → stats.languages로 수정.
fetchStats() 불필요한 래퍼 제거
data.data || {}를 response.json() 직접 반환으로 단순화.
tags 타입 불일치 (String vs Array)
SQLite에 저장된 "git,revert" 문자열을 .map()으로 처리 시 오류.
→ normalizeSnippet() 헬퍼 추가, tags를 배열로 정규화.
스니펫 생성 시 tags 직렬화 누락
폼 tags 배열을 백엔드 전송 전 .join(',')으로 재결합 — DB 저장 형식 맞춤.
snippet.id 타입 통일 (Number → String)
card.getAttribute('data-id')는 문자열이지만 API 반환값은 숫자.
→ normalizeSnippet에서 String(s.id)로 통일하여 === 비교 안전화.

🚀 실행 방법

1. 의존성 설치

python3.9 -m pip install fastapi "uvicorn[standard]"

2. 서버 시작

# snip 패키지 루트에서 실행
cd /Users/hongpaul/develop/project-persnol/snip
python3.9 -m uvicorn snip.web.app:app --host 127.0.0.1 --port 8000

3. 브라우저 접속

http://127.0.0.1:8000          # 웹 앱 메인
http://127.0.0.1:8000/docs     # Swagger UI (자동 생성)
http://127.0.0.1:8000/redoc    # ReDoc API 문서

참고: 웹 앱과 CLI(snip 명령어)는 동일한 ~/.snip/snip.db를 공유합니다. CLI로 추가한 스니펫이 웹에서 즉시 표시되며 반대도 동일합니다.
